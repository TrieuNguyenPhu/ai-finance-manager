# Implement feature prompt

Implement: <FEATURE>

Acceptance criteria:
- <CRITERION_1>
- <CRITERION_2>

Workflow:
1. Read the relevant rules and inspect existing implementation, contracts, call sites, and tests.
2. State assumptions and a concise plan.
3. Implement the smallest coherent change.
4. Add or update focused tests.
5. Run the narrowest relevant checks, then broader checks only when justified.
6. Inspect the final diff.

Do not:
- modify unrelated files;
- invent APIs, schemas, events, or environment variables;
- add dependencies without explaining why existing options are insufficient;
- bypass validation, security, typing, lint, or tests.

Final response:
- summary;
- files changed;
- commands/tests and outcomes;
- assumptions;
- remaining risks.
